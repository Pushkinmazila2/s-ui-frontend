# Git Integration для S-UI - Финальная версия ✅

## Обзор изменений

Добавлена полная интеграция с Git репозиториями (GitHub, GitLab, Gitea) для автоматического резервного копирования конфигурации и базы данных S-UI.

## Реализованные функции

### ✅ Основные возможности

1. **Включение/выключение синхронизации**
   - Простой переключатель для активации функции

2. **Выбор Git провайдера**
   - GitHub
   - GitLab
   - Gitea

3. **Настройка репозитория**
   - URL репозитория
   - Ветка (по умолчанию `main`)
   - Токен доступа (скрытый ввод)

4. **Режимы синхронизации**
   - **Автоматическая синхронизация** - по расписанию с настраиваемым интервалом (в секундах)
   - **Синхронизация при изменении** - автоматический push при изменении конфигурации
   - **Ручная синхронизация** - кнопки Push Now / Pull Now

5. **Выбор данных для синхронизации**
   - ✅ Sync Config - конфигурация Sing-Box
   - ✅ Sync Database - база данных S-UI

6. **Дополнительные функции**
   - Тест подключения к репозиторию
   - Отображение времени последней синхронизации
   - Индикаторы загрузки для всех операций
   - Уведомления об успехе/ошибке

**Структура данных:**
```typescript
{
  enable: boolean,         // Включена ли синхронизация
  provider: string,        // 'github' | 'gitlab' | 'gitea'
  repoUrl: string,        // URL репозитория
  branch: string,         // Ветка
  token: string,          // Токен доступа

  syncDb: boolean,        // Синхронизировать базу данных
  lastSync: number        // Unix timestamp последней синхронизации
}
```

- `POST /api/gitSyncTest` - тест подключения
- `POST /api/gitSyncPush` - ручной push

## Инструкция по использованию

### Шаг 1: Создание Git репозитория

1. Создайте новый приватный репозиторий на GitHub/GitLab/Gitea
2. Назовите его, например: `s-ui-backup`

### Шаг 2: Получение токена доступа

**GitHub:**
1. Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token (classic)
3. Выберите scopes:
   - ✅ `repo` (Full control of private repositories)
4. Скопируйте токен (начинается с `ghp_`)

**GitLab:**
1. User Settings → Access Tokens
2. Add new token
3. Выберите scopes:
   - ✅ `api`
   - ✅ `read_repository`
   - ✅ `write_repository`
4. Скопируйте токен (начинается с `glpat-`)

**Gitea:**
1. Settings → Applications → Manage Access Tokens
2. Generate Token
3. Выберите permissions:
   - ✅ `repository` (Read and Write)
4. Скопируйте токен

### Шаг 3: Настройка в S-UI

1. Откройте **Settings → GIT**
2. Включите переключатель **Enable**
3. Выберите **Provider** (GitHub/GitLab/Gitea)
4. Введите **Repository URL**:
   - GitHub: `https://github.com/username/s-ui-backup.git`
   - GitLab: `https://gitlab.com/username/s-ui-backup.git`
   - Gitea: `https://gitea.example.com/username/s-ui-backup.git`
5. Укажите **Branch** (обычно `main` или `master`)
6. Вставьте **Access Token**
7. Настройте режимы синхронизации:
   - **Auto Sync** - автоматическая синхронизация по расписанию
   - **Sync Interval** - интервал в секундах (3600 = 1 час)
   - **Sync on Change** - автоматический push при изменении конфигурации
8. Выберите что синхронизировать:
   - **Sync Config** - конфигурация Sing-Box
   - **Sync Database** - база данных S-UI
9. Нажмите **Save**
10. Нажмите **Test Connection** для проверки

### Шаг 4: Использование

**Автоматическая синхронизация:**
- Если включен **Auto Sync**, бэкенд будет автоматически делать push каждые N секунд
- Если включен **Sync on Change**, push будет выполняться при каждом изменении конфигурации

**Ручная синхронизация:**
- **Push Now** - отправить текущие данные в Git
- **Pull Now** - получить данные из Git и применить их

## Технические детали

### Аутентификация

Фронтенд использует **session cookie** для аутентификации с бэкендом.
Все запросы идут через `/api/` endpoints (не `/apiv2/`).

### Хранение данных

Конфигурация Git хранится в базе данных бэкенда.
При монтировании компонента данные загружаются через `GET /api/gitSyncConfig`.

### Сохранение изменений

Изменения НЕ сохраняются автоматически.
Пользователь должен нажать кнопку **Save** для сохранения конфигурации.

### Обработка ошибок

Все API запросы обрабатываются через `HttpUtils`:
- Автоматические уведомления об успехе/ошибке
- Обработка невалидного логина
- Структурированный ответ `{ success, msg, obj }`

**Request:**

**Response:**
```json
{
  "success": true,
  "msg": "",
  "obj": { /* обновленная конфигурация */ }
}
```

### POST /api/gitSyncTest

Тестирует подключение к Git репозиторию.

### POST /api/gitSyncPush

Выполняет push в Git репозиторий.
Должен учитывать настройки `syncConfig` и `syncDb`.

### POST /api/gitSyncPull

Выполняет pull из Git репозитория.
Должен учитывать настройки `syncConfig` и `syncDb`.

## Логика бэкенда

### Auto Sync (автоматическая синхронизация)

Когда `autoSync = true`:
1. Запустить фоновый процесс
2. Каждые `syncInterval` секунд выполнять push
3. Учитывать настройки `syncConfig` и `syncDb`
4. Обновлять `lastSync` после успешного push

### Sync on Change (синхронизация при изменении)

Когда `syncOnChange = true`:
1. Отслеживать изменения в конфигурации Sing-Box
2. При изменении выполнять push
3. Учитывать настройки `syncConfig` и `syncDb`
4. Обновлять `lastSync` после успешного push

### Что синхронизировать

- Если `syncConfig = true` - включить в commit файлы конфигурации
- Если `syncDb = true` - включить в commit файл базы данных
- Можно включить оба варианта одновременно

## Известные проблемы и решения

### Проблема: Настройки не сохраняются

**Решение:** Убрано автоматическое сохранение. Теперь пользователь должен нажать кнопку "Save".

### Проблема: Синхронизируется все, игнорируя настройки

**Решение:** Бэкенд должен проверять флаги `syncConfig` и `syncDb` перед добавлением файлов в commit.

### Проблема: Нет опции "синхронизация при изменении"

**Решение:** Добавлено поле `syncOnChange` для автоматического push при изменении конфигурации.
